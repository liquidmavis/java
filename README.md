pull request的目的：和原仓库的开发者交互，commit自己的一些改动。

1、登录自己的github账号

2、fork项目

3、将fork的项目克隆（clone）到本地，（可以使用一些git工具，比如Github Desktop、SourceTree等）

4、在clone的项目中修改内容，并commit，然后推送到远端

5、在fork的原仓库打开Pull Request栏，提交修改信息，等待原作者同意就可以了
